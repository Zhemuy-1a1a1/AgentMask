# Sample Inputs / 示例输入

> 每个 Agent 的真实使用场景示例。用户会实际输入什么内容？以下是3个核心 Agent 的具体场景输入。

---

## 小红书运营助手

### 场景一：母婴营养师个人号冷启动
```
账号定位：二线城市母婴营养师，服务0-3岁宝宝家庭，主打"低成本、可执行、科学喂养"
选题方向：夏季宝宝不爱吃饭怎么办（辅食阶段）
目标受众：25-35岁新手妈妈
内容目标：提高收藏和私信咨询
输出要求：给我10个标题备选、1个正文结构（分段提纲）、15个标签
```

### 场景二：本地探店号转化（咖啡馆）
```
账号定位：成都本地探店号，已有1.2万粉丝，目标"周末到店转化"
选题方向：春熙路附近适合拍照+办公的咖啡馆清单
额外约束：突出价格区间、交通便利度、适合人群
目标：带动评论区问店名和收藏地图
输出要求：10个标题备选、1个正文结构、10个标签
```

## 客服回复助手

### 场景一：物流延迟投诉（电商）
```
客户原话："你们这物流也太慢了吧？我明天要送人，现在还没到，真离谱！"
场景：大促期间，华东地区暴雨导致干线延误；订单显示已到本地站点
客服目标：安抚情绪 + 提供可验证信息 + 降低退款率
业务规则：可发5-15元无门槛券；超过承诺时效48小时可申请优先退款
回复风格：真诚、不甩锅、给时间节点
输出要求：主回复（安抚+解释+方案）/ 备用回复 / 是否需要升级人工判断
```

### 场景二：SaaS功能故障+退款诉求
```
客户原话："导出功能一直报错，我们今天必须交报表。你们不解决我就退款。"
场景：B端SaaS专业版客户；故障疑似由浏览器插件冲突引起
业务规则：P1工单15分钟响应；可提供1个月服务时长补偿
回复风格：专业、简洁、可执行
输出要求：主回复 / 备用回复 / 升级判断（是否转售后专员）
```

## 报价单Agent

### 场景一：企业官网改版+多语言
```
客户需求描述：
"我们想重做官网，品牌形象要更国际化。需要中英文切换、产品中心、案例页、询盘表单、SEO基础优化，最好两个月上线。后续希望有人维护。"
补充信息：
- 预算区间15-25万
- 需开票
- 希望分阶段付款
- 客户内部素材交付可能延迟
输出要求：报价单结构（模块拆分）+ 具体数字（单项与总价）+ 付款条款（节点比例）
```

### 场景二：小程序定制开发（会员+预约）
```
客户需求描述：
"我们要做一个小程序，用户能买课包、约团课、到店核销，还要看剩余次数。希望有后台给店长看数据，后面可能接企业微信。"
补充信息：
- 已有视觉稿50%
- 需求可能迭代
- 要求上线后3个月质保
商务条件：接受里程碑付款；希望看到明细报价和可选项
输出要求：标准版报价 + 可选增强项 + 实施排期 + 风险条款
```
