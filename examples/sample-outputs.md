# Sample Outputs / 示例输出

> 每个 Agent 的期望输出摘要。不是完整输出，而是关键特征描述。

---

## 小红书运营助手

### 场景一（母婴营养师）输出特征
- **标题**：10个备选，含数字/场景词/结果词（如"3步""夏天""不追着喂"）
- **正文结构**：痛点开场 → 原因拆解 → 操作步骤（周一到周日）→ 避雷提醒 → 互动引导
- **标签**：10-15个，含核心垂类标签（母婴/辅食）+ 长尾问题标签
- **语气**：专业安抚型、可执行清单化

### 场景二（咖啡馆探店）输出特征
- **标题**：10个，体现"季节限定/城市感/口味记忆"（如"春熙路旁""茉莉荔枝"）
- **正文结构**：成分亮点 → 口感分层 → 适配场景 → 下单方式
- **标签**：含地域标签（上海甜品/成都探店）+ 品类标签
- **互动引导**：评论关键词触发私信地图

## 客服回复助手

### 场景一（物流投诉）输出特征
- **主回复**：共情道歉 → 事实说明（提供物流单号）→ 补偿方案（5-15元券）→ 承诺时间节点
- **备用回复**：更短，适合移动端快速发送，语气温和
- **升级判断**：明确——出现"投诉平台/监管/辱骂/二次催促未解决"即升级

### 场景二（SaaS故障）输出特征
- **主回复**：道歉 → 立即确认问题 → 提供临时方案（如导出为CSV）→ 补偿（1个月时长）→ 承诺跟进
- **备用回复**：简洁版，给出"联系客服走P1通道"
- **升级判断**：涉及赔偿金额、差评威胁、重复投诉 → 转售后专员

## 报价单Agent

### 场景一（官网改版）输出特征
- **报价单结构**：项目范围 → 分项费用 → 排除项 → 交付物 → 里程碑（8-10周）
- **费用明细**：每项单价/小计/总计/税费说明，总价控制在预算区间内
- **付款条款**：`40%-40%-20%`（签约/中期/验收），对应明确里程碑
- **风险条款**：需求变更定价机制、延期责任、质保期

### 场景二（小程序开发）输出特征
- **报价单结构**：业务目标 → 功能清单（C端+B端后台）→ 技术依赖 → 基础包报价 → 可选增强项
- **实施排期**：12周，里程碑节点明确
- **付款条款**：含预付款比例、验收节点、质保期
- **风险条款**：需求变更处理机制、素材延迟处理
