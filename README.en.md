# One-File Agent Packs (English Guide)

> One file, one agent, ready to import and use.

[Back to Home](./README.md) · [Quick Start](./QUICKSTART.en.md) · [Upgrade Path](./UPGRADE.md) · [Agent Index](./agents/INDEX.md)

---

## What is this?
This repo turns common real-world tasks into **one-file agent packs**.

You do not need a complex framework to get started.
You do not need to study prompt theory first.
You just:
1. pick a pack
2. open the markdown file
3. copy it into your AI agent instructions
4. start using it

---

## Who is this for?
- People building their own custom AI agents
- People doing AI content, AI services, or AI monetization
- People turning prompts into real deliverables
- People who want to try the open-source version first before upgrading

---

## What is included right now?
This repo currently contains the **lightweight open-source layer**.
It is meant for:
- quick testing
- fast onboarding
- sharing
- lead generation
- early workflow validation

---

## How to use this
### Step 1: Choose a pack
Pick the scenario closest to your task.

### Step 2: Copy the whole file
Copy the entire markdown file.

### Step 3: Paste it into your AI agent instructions
This can be:
- system prompt
- custom instructions
- role definition
- project-level prompt setup

### Step 4: Start with the input template
If you are unsure how to ask, use:
[Sample Inputs](./examples/sample-inputs.md)

### Step 5: Adapt the output to your real workflow
The pack gives you a usable first version.
Then you adapt tone, business rules, or internal context as needed.

For a more detailed guide, see:
[Quick Start](./QUICKSTART.en.md)

---

## Why this repo exists
Most people do **not** need more theory.
They need something they can:
- import quickly
- understand quickly
- adapt quickly
- use for actual work

That is why this repo keeps the structure intentionally simple:
**one scenario → one pack → one markdown file**

---

## What is usually inside each pack?
A pack usually includes:
- role definition
- core capabilities
- input template
- output structure
- basic rules
- usage notes

So this is not just a random prompt line.
It is a packaged one-file agent starter.

---

## Need something stronger?
This repo is only the public lightweight layer.
If you want a stronger version for real long-term use, the upgrade path is now:
- **Standard version**
- **Advanced (custom) version**

Even after upgrading, the core product direction stays the same:
**one agent = one core `.md` file**.

See:
[UPGRADE.md](./UPGRADE.md)
