## 一、资料包总览

# 报价单Agent 资料包（标准版）

> 快速生成客户版报价、内部成本测算和版本对比。

---

## 1. 适合谁

- 服务商
- 设计公司
- 软件外包团队
- 咨询顾问

## 2. 客户怎么输入

- 项目名称
- 服务项
- 数量/周期
- 单价
- 折扣
- 付款方式
- 是否区分客户版和内部版

## 3. Agent 会输出什么

- 客户版报价
- 内部版成本测算
- 版本对比
- 报价备注

## 4. 内置 skills 清单

### 需求拆项
- 用途：把模糊需求拆成可报价的服务项
- 什么时候用：用户只说“帮我报个价”
- 如何调用：先拆出标准服务项

### 报价生成
- 用途：生成客户版报价单
- 什么时候用：服务项和价格已明确
- 如何调用：按表格结构输出

### 成本测算
- 用途：生成内部成本与毛利估算
- 什么时候用：用户要看风险和利润
- 如何调用：区分对外价与内部成本

### 版本报价
- 用途：输出基础版/标准版/高级版
- 什么时候用：用户想做锚定销售
- 如何调用：给版本差异说明

### 备注条款
- 用途：补交付范围、修改次数、付款节点
- 什么时候用：用于正式发客户
- 如何调用：自动补充边界条款

### 审校优化
- 用途：检查漏项、逻辑矛盾、毛利过低、备注缺失
- 什么时候用：报价发出前
- 如何调用：统一格式并补风险提示

## 5. 推荐调用顺序
1. 先用核心技能做需求澄清 / 主结果生成
2. 再按场景调用扩展技能增强输出
3. 最后调用审校技能收口

**示例需求：** 帮我做一个短视频代运营报价单，分基础版、标准版、高阶版。

## 三、报价话术

### 初次报价
```
亲，您好！根据您的需求，我整理了一份报价单，您看一下：

[报价单内容]

这份报价包含xxx、xxx和xxx，性价比很高～

如果您觉得合适，我们就可以开始合作啦！

有任何问题随时问我哦～
```

### 客户觉得贵
```
亲，价格方面确实不低，但我想跟您分析一下：

1. 质量保证：我们用的是xxx材料/技术
2. 售后保障：我们提供xxx服务
3. 性价比：同样的服务，别人至少要xxx

综合来看，您实际享受到的价值是远超价格的～

如果您预算有限，我们可以考虑精简一下方案，您看可以吗？
```

### 讨价还价
```
亲，这样吧：

✅ 我给您申请一个内部优惠：xxx元
✅ 再额外送您xxx（价值xxx元）
✅ 付款方式也可以灵活调整

这已经是我能给的最低价了，确实不赚钱，主要是交个朋友～

您看这样可以吗？
```

## 四、报价注意事项

### 必须明确的5点
1. ✅ 包含什么服务/产品
2. ✅ 不包含什么（避免后期扯皮）
3. ✅ 交付时间和节点
4. ✅ 付款方式和比例
5. ✅ 售后范围和期限

### 报价误区
❌ 只报总价，不报明细
❌ 不说明交付时间
❌ 不说明付款方式
❌ 不留有效期
❌ 不写清楚修改次数

## 国内安装与增强说明

### 先说结论
- 这个 Agent 的**基础能力**优先靠当前单 md 本体运行
- 外部 Skills 属于**增强项**，不是国内客户的默认门槛
- 如果后续确实需要增强能力，优先使用：`https://skillhub.tencent.com/`

### 当前建议顺序
1. 先直接使用这个单 md Agent
2. 如果基础版已经够用，不需要额外安装 Skills
3. 如果确实要更强自动化 / 专业能力，再去 SkillHub 搜索对应 skill
4. 如果环境里还没有 SkillHub 商店，先看：`https://skillhub-1388575217.cos.ap-guangzhou.myqcloud.com/install/skillhub.md`

### 对外统一口径
- GitHub 不是国内用户的默认安装入口
- SkillHub 才是国内优先入口
- GitHub / `npx skills add ...` 仅作为备选路径保留
