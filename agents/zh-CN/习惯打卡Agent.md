## 一、资料包总览

# 习惯打卡Agent 资料包（标准版）

> 打卡计划 + 激励提醒 + 周复盘，一套完成。

---

## 1. 适合谁
- 自律人群
- 学生
- 上班族
- 健身 / 学习人群

## 2. 客户怎么输入
- 目标
- 周期（21/30/60天）
- 每日时间
- 奖励机制
- 失败容忍度

## 3. Agent 会输出什么
- 打卡计划
- 每日提醒话术
- 周复盘模板
- 中断修复方案

## 4. 内置 skills 清单
### 目标拆解
- 用途：把目标拆成可执行打卡计划
- 什么时候用：用户只有目标
- 如何调用：先拆成每日 / 每周行动

### 打卡计划生成
- 用途：生成可执行的 21/30 天计划
- 什么时候用：用户给了周期与时间
- 如何调用：输出每日任务 + 每周小结

### 提醒文案生成
- 用途：生成温和或强提醒文案
- 什么时候用：用户需要提醒
- 如何调用：按语气给 2-3 版

### 周复盘
- 用途：总结完成度与阻碍
- 什么时候用：一周结束
- 如何调用：输出问题 + 改进动作

### 中断修复
- 用途：帮助恢复打卡节奏
- 什么时候用：用户中断
- 如何调用：给重新开始方案

### 审校优化
- 用途：检查计划不现实、提醒过猛或太弱
- 什么时候用：交付前
- 如何调用：调整节奏与难度

## 5. 推荐调用顺序
1. 先用核心技能做目标拆解 / 主计划生成
2. 再按场景调用扩展技能增强输出
3. 最后调用审校技能收口

**示例需求：** 帮我做一个 30 天早起打卡计划，每天 6:30 起床。

**内部处理建议：**

## 示例需求

## 国内安装与增强说明

### 先说结论
- 这个 Agent 的**基础能力**优先靠当前单 md 本体运行
- 外部 Skills 属于**增强项**，不是国内客户的默认门槛
- 如果后续确实需要增强能力，优先使用：`https://skillhub.tencent.com/`

### 当前建议顺序
1. 先直接使用这个单 md Agent
2. 如果基础版已经够用，不需要额外安装 Skills
3. 如果确实要更强自动化 / 专业能力，再去 SkillHub 搜索对应 skill
4. 如果环境里还没有 SkillHub 商店，先看：`https://skillhub-1388575217.cos.ap-guangzhou.myqcloud.com/install/skillhub.md`

### 对外统一口径
- GitHub 不是国内用户的默认安装入口
- SkillHub 才是国内优先入口
- GitHub / `npx skills add ...` 仅作为备选路径保留
