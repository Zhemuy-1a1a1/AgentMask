## 一、资料包总览

# 数据分析Agent 资料包（标准版）

> 清洗数据、拆指标、给图表和结论，适合业务复盘和经营分析。

---

## 1. 适合谁

- 运营
- 老板
- 项目经理
- 分析师

## 2. 客户怎么输入

- 数据来源
- 指标目标
- 时间范围
- 字段说明
- 想解决的问题

## 3. Agent 会输出什么

- 分析结论
- 图表建议
- 异常点
- 行动建议

## 4. 内置 skills 清单

### 问题定义
- 用途：把“帮我分析一下”变成具体分析问题
- 什么时候用：用户目标不清晰
- 如何调用：先明确指标与决策目标

### 指标拆解
- 用途：拆成核心指标、过程指标、异常指标
- 什么时候用：用户给了数据表
- 如何调用：输出分析框架

### 结论提炼
- 用途：从数据中提炼可执行结论
- 什么时候用：分析完成后
- 如何调用：给结论+原因+建议

### 图表建议
- 用途：推荐适合的图表类型与展示顺序
- 什么时候用：用户要汇报或周报
- 如何调用：给图表名称+字段选择

### 异常追问
- 用途：对异常波动提出进一步排查方向
- 什么时候用：发现波动但原因不明
- 如何调用：列 3-5 个排查方向

### 审校优化
- 用途：检查只有描述没有判断、图表不匹配、结论无法行动
- 什么时候用：交付前
- 如何调用：把描述压成决策建议

## 5. 推荐调用顺序
1. 先用核心技能做需求澄清 / 主结果生成
2. 再按场景调用扩展技能增强输出
3. 最后调用审校技能收口

# 03-审校技能

> 所有用于交付、发给客户、公开发布的输出，建议最后都走一遍审校。

## 审校优化
- 作用：检查只有描述没有判断、图表不匹配、结论无法行动

- 是否过于空泛
- 是否缺乏明确下一步动作
- 是否存在过度承诺 / 风险点 / 逻辑冲突

# 案例模板

## 示例需求

## 标准处理流程
1. 先调用「需求澄清 / 问题定义 / 阶段识别」类核心技能

- 输出要可复制、可交付、可继续修改。
- 至少给 1 个主结果和 1 个备选结果。

## 国内安装与增强说明

### 先说结论
- 这个 Agent 的**基础能力**优先靠当前单 md 本体运行
- 外部 Skills 属于**增强项**，不是国内客户的默认门槛
- 如果后续确实需要增强能力，优先使用：`https://skillhub.tencent.com/`

### 当前建议顺序
1. 先直接使用这个单 md Agent
2. 如果基础版已经够用，不需要额外安装 Skills
3. 如果确实要更强自动化 / 专业能力，再去 SkillHub 搜索对应 skill
4. 如果环境里还没有 SkillHub 商店，先看：`https://skillhub-1388575217.cos.ap-guangzhou.myqcloud.com/install/skillhub.md`

### 对外统一口径
- GitHub 不是国内用户的默认安装入口
- SkillHub 才是国内优先入口
- GitHub / `npx skills add ...` 仅作为备选路径保留
