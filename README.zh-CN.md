## 这是什么？
这是一个把常见业务场景整理成 **单文件 Agent 包** 的仓库。

你不需要搭很复杂的框架，也不需要先研究很多 prompt 理论。
你只需要：
1. 找到一个适合你的 Agent 包
2. 打开对应的 `.md` 文件
3. 复制进自己的 Agent 指令区
4. 开始输入需求

就这么简单。

## 怎么用（小白版）
### 第一步：选一个 Agent
比如你想做小红书，就先打开：
[小红书运营助手](./agents/zh-CN/小红书运营助手.md)

### 第二步：复制整个文件内容
把这个 `.md` 文件的内容完整复制出来。

### 第三步：粘贴到你的 Agent 指令区
常见放法：
- 系统提示词
- Instructions
- 自定义 Agent 的角色描述
- 项目级提示词

### 第四步：按输入模板提需求
如果你不知道怎么提，先看：
[示例输入](./examples/sample-inputs.md)

### 第五步：拿到结果后再微调
Agent 会先给你一版可用结果，
你再按自己的业务口径、语气、限制去微调。

更详细的入门说明见：
[快速开始](./QUICKSTART.zh-CN.md)

## 这套东西为什么有用
很多人卡住，不是不会用 AI，
而是不会把一个场景整理成：
- 明确角色
- 清晰输入
- 稳定输出
- 基础规则
- 可交付文件

这个仓库解决的就是这个问题。

它不是让你“研究”，
而是先让你“跑起来”。

---

## 你会在每个包里看到什么
通常一个包里会有：
- 角色定义
- 核心能力
- 输入模板
- 输出结构
- 基础规则
- 使用说明

也就是说，
它不是一句孤零零的 prompt，
而是一份已经整理过的单文件 Agent 包。

## 如果你想升级
当前仓库是 **标准公开层**。
如果你想拿去：
- 真正长期使用
- 更少重复训练
- 更低 API 成本
- 更贴近国内模型 / 国内源环境
- 做深度适配

那后续升级方向现在统一为：
- **增强优化版**
- **进阶（定制）版**

不管升级到哪一层，核心仍然建议保持：
**一个 Agent = 一个单 md 文件。**

详见：
[升级路径](./UPGRADE.md)
